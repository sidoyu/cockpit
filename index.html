cockpit — Claude Code, 세팅 없이 바로 쓰기

한 사람이 오래 다듬은 Claude Code 운영 환경을 그대로 담은 설치판입니다. 아래 버튼으로 받아서 더블클릭하면 끝납니다. Windows 기준입니다.

먼저 3가지만 알아두세요

  • 개인 PC에서, 업무 자료·환자정보 없이 쓰는 개인 학습용 도구입니다(회사 공식 배포물 아님).
  • Claude가 일일이 묻지 않고 스스로 진행하는 강력한 설정입니다. 위험 명령 차단 장치가 있지만 완전하지는 않습니다.
  • 본인 Claude 구독으로 로그인해서 씁니다. 입력 내용은 Anthropic에 전송됩니다.

체크하면 아래 다운로드 버튼이 활성화됩니다.

자세한 경계는 아래 사용 경계 전문에 있습니다.

⬇ 설치 파일 받기 Cockpit-Install.cmd · Windows

이 파일은 코드서명이 없어 “알 수 없는 게시자” 경고가 뜰 수 있습니다. 반드시 이 페이지의 버튼으로 받으세요. 꼼꼼히 검증하려면 아래 모든 파일·검증.

  1. 위 버튼을 눌러 파일을 받습니다.
  2. 받은 파일을 더블클릭합니다. 검은 창이 열리고 알아서 설치합니다(인터넷 필요 · 수 분 소요). 파란 경고 창(“Windows의 PC 보호”)이 뜨면 [추가 정보] → [실행]을 누르세요. 경고 없이 바로 진행되는 PC도 있는데 둘 다 정상입니다. 설치가 거의 끝나면 작은 설정 창이 한 번 뜹니다 — 개인 PC·비업무 전용 동의(필수 체크)와, 원하면 기억 자동정리용 API 키를 여기서 정합니다. 세션 기록 대시보드는 이 창과 무관하게 기본 부속으로 자동 설치됩니다(설치≠기동 — 켜기는 아이콘). 잘 모르면 [건너뛰기]를 눌러도 됩니다 — 아래 /cockpit-setup 마법사가 나중에 그대로 물어봅니다.
  3. 설치가 끝나면 바탕화면의 “Claude (cockpit)” 아이콘을 더블클릭합니다.
  4. 검은 창에 /login 을 입력하고 Enter. 브라우저가 열리면 본인 Claude 계정으로 승인합니다.
  5. 같은 창에 /cockpit-setup 을 입력합니다. 설치 창에서 이미 정한 항목(동의·API 키)은 건너뛰고, 남은 세부 설정을 마법사가 안내합니다(설치 창에서 [건너뛰기]했다면 여기서 전부 물어봅니다). 대시보드는 자동 설치되며, 설치에 실패했을 때만 여기서 재시도를 안내합니다. 휴대폰·claude.ai에서 이어 쓰는 원격 조종을 켜려면, 안내에 따라 창을 한 번 껐다 다시 켜면 됩니다. 끝입니다.

팁: 추론 강도는 /effort 에서 xhigh를 고르세요(계정 한도에 따라 high로 남을 수 있고 그것도 정상).

더 알아보기

무엇이 깔리나

작업 규율(행동층)

자율 진행 경계, 두 순간 자기점검, 읽기/검색 위생. 설치 마법사가 ~/.claude/CLAUDE.md로 적용.

메모리 시스템

파일 기반(md + python 훅). 세션 시작 시 상태 주입, 종료 시 기억 후보 추출, 인덱스 자동 재생성. 자동 추출(외부 송신)은 개인 API 키 옵트인 — 설치 창(또는 /cockpit-setup 마법사)에서 등록하며 미등록 시 기억은 수동으로만 쌓입니다(키 사용량은 정액 구독과 별개로 과금, 콘솔에서 한도 설정 권장).

안전망

파괴/비가역 명령 금지목록(deny-list), 긴급정지(kill switch), 감사 로그.

세션 기록 대시보드(기본 부속)

세션 로그를 이 PC의 브라우저 창으로 열람(바탕화면 'Cockpit Dashboard' 아이콘 또는 Cockpit-Dashboard.cmd 더블클릭 · 창 닫으면 꺼짐). 뷰어는 기본 설치(설치≠기동)·기본 로컬 전용, 타기기 원격 열람은 고급·옵트인.

설치 마법사

기본 사용자·메모리 디렉터리·선택 기능을 안내하고 충돌 검사·dry-run·롤백을 제공.

문제가 생겼을 때
  • 창에 “WSL 준비가 필요합니다” 안내가 나오면 — 시작 메뉴에서 PowerShell을 찾아 우클릭 → 관리자 권한으로 실행한 뒤, 창에 표시된 명령만 그대로 입력하고(다른 명령을 검색해 입력하지 마세요), 끝나면 다시 설치 파일을 더블클릭하세요. 설치기는 스스로 관리자 권한을 올리지 않습니다.
  • 빨간 글씨로 “배포판 'cc-cockpit' 이 이미 등록되어 있습니다” — 이미 설치된 PC입니다(안전장치가 막은 것·기존 설치 무사). 정말 처음부터 다시 깔려면 아래 복구·재설치 Cockpit-Repair.cmd를 쓰세요 — 배포판 안의 기억·설정이 지워집니다(이중 확인 후 진행).
  • 설치 중 실패 — 창이 바로 닫히지 않고 이유를 보여줍니다. 대부분 인터넷 연결 문제이니 다시 실행해 보고, 안 되면 창에 보이는 화면을 사진으로 찍어 배포자에게 보내주세요.
  • 완전 제거 — Cockpit-Uninstall.cmd 더블클릭(cc-cockpit 배포판만 삭제 · 다른 WSL 배포판은 건드리지 않음).

점검·긴급 정지

상태 점검

Claude 창에 입력 — 설치 상태·원격 ON/OFF 확인:

/cockpit-doctor

긴급 정지

자동 진행을 즉시 멈춤(파일 하나 생성):

touch ~/.claude/CC_KILL_SWITCH

원격 끄기

먼저 미리보기, 그다음 실제 비활성화(설치된 플러그인 폴더에서 실행):

bash plugin/dashboard/disable-remote.sh
bash plugin/dashboard/disable-remote.sh --apply

경로가 안 맞으면 “file not found”가 납니다 — 정확한 절대경로는 /cockpit-doctor가 안내합니다.

플러그인만 제거하려면 /plugin uninstall + 설치 마법사의 롤백 안내. 자세한 절차는 거버넌스 문서 6장.

세션 기록 대시보드 켜기 (로컬 · 기본 부속)

대시보드 뷰어는 기본 부속으로 자동 설치됩니다(설치≠기동 — 자동시작·포트 개방 없음). 바탕화면의 'Cockpit Dashboard' 아이콘(설치기가 만들어 둠)이나 Cockpit-Dashboard.cmd를 더블클릭해 세션 기록을 이 PC의 브라우저 창으로 봅니다 (설치가 안 돼 있으면 인터넷 연결 후 아이콘을 다시 누르거나 /cockpit-setup으로 재시도) (아래 다운로드 표). 서버 개념을 다룰 필요 없이, 새로 켠 창을 닫으면 서버도 함께 꺼집니다(정확한 조건은 아래).

  • 이 PC 전용(로컬). 대시보드는 127.0.0.1에만 열려, 기본 설정에서는 다른 기기에서 접속되지 않습니다.
  • 세션 기록에는 민감한 내용이 담길 수 있습니다. 열람 창을 방치하지 말고, 끝나면 창을 닫으세요. 뷰어 idle-exit(무요청 시 자가 종료)가 백스톱입니다.
  • 종료 조건은 상황에 따라 다릅니다. ① Edge가 있고 이번에 새로 켠 경우 — 전용 앱 창을 닫으면 서버도 종료(.cmd가 창 종료를 감지). ② 이미 켜져 있던 서버를 다시 연 경우 — 이 창을 닫아도 서버는 안 꺼지니 /cockpit-doctor로 확인·종료. ③ Edge가 없으면 기본 브라우저로 열리고, 끝난 뒤 .cmd 창에서 안내대로 키를 눌러 종료합니다.
  • 상태 확인·강제 끄기는 /cockpit-doctor가 안내합니다(--app 없이 일반 창으로 직접 열면 창 종료 감지가 안 되니 .cmd로만 여세요).

폰·노트북 등 다른 기기에서 열람하려는 경우는 아래 원격 대시보드(고급·선택)를 따르세요 — 위험도가 높아 기본 꺼짐입니다.

사용 경계 전문 (설치 전 권장)
  • 권한 확인 생략(bypass) = 켜짐. Claude가 명령 실행 전 일일이 묻지 않고 진행합니다(대표적 파괴/비가역 명령은 금지목록으로 차단하지만 완전한 보호는 아닙니다).
  • 외부 송출. 입력 내용이 모델 응답을 위해 Anthropic에 전송되고, 일부 기능(메모리 자동 추출·선택적 대시보드 요약)은 추가 외부 송신을 수반할 수 있습니다.
  • 원격 대시보드 = 기본 꺼짐(opt-in). 켜면 개인 VPN 안의 다른 기기에서 호스트를 조종할 수 있습니다(가장 위험).
  • 개인 PC에서, 비업무·비민감 데이터로만. 환자정보·개인정보(PII)·기밀을 절대 입력하지 마세요.
  • 안전장치는 보조일 뿐이며 최종 책임은 사용자에게 있습니다. 회사 보안정책과 충돌하면 회사 정책이 우선합니다.

미서명 배포와 검증에 대해 (정직 고지)

이 배포본은 코드서명이 없어서(unsigned) Windows가 “알 수 없는 게시자” 경고를 띄울 수 있습니다(Get-AuthenticodeSignature가 NotSigned로 나오는 것이 정상 · .cmd 첫 실행이 .ps1보다 경고가 부드럽다는 보장은 없습니다). 자동 대조되는 것은 .cmd가 받아오는 Install-Cockpit.ps1과 그 스크립트가 받는 이미지(cockpit-wsl.tar.gz·import 전 대조, 불일치 시 삭제·중단)입니다 — 두 파일의 핀 해시가 .cmd/스크립트 안에 박혀 있습니다. 단, .cmd 자체는 신뢰의 출발점이라 자동 검증되지 않습니다: .cmd가 변조되면 내장 핀 해시도 함께 변조될 수 있으므로(미서명+체크섬 정책의 구조적 한계), 공식 다운로드 링크에서 받고, 확실히 하려면 받은 .cmd의 Get-FileHash 값을 위 표의 SHA-256과 직접 대조하세요(생략하면 다운로드 채널을 신뢰하는 셈).

설치기는 구형 WSL이면 .tar로 풀어 스스로 재시도하고, 그래도 안 되면 이유를 보여주며 멈춥니다(자동으로 다른 스크립트로 넘어가지 않습니다). 골든 이미지 경로가 맞지 않을 때 쓰는 대안 부트스트랩이 스테이지드(Install-Cockpit-Staged.ps1)입니다 — 공식 베이스 rootfs를 검증해 import 후 라이브 프로비저닝(직접 골라 실행). 고급(수동): Install-Cockpit.ps1을 직접 받아 Get-FileHash … -Algorithm SHA256로 위 표의 해시와 대조한 뒤 powershell -NoProfile -ExecutionPolicy Bypass -File .\Install-Cockpit.ps1로 실행할 수도 있습니다.

원격 대시보드 (고급 · 선택 · 기본 꺼짐)

휴대폰·노트북 등 같은 개인 VPN(예: Tailscale)에 묶인 다른 기기에서 세션 로그를 보거나 세션을 조종하는 선택 기능입니다. 이 패키지에서 가장 위험한 기능이라 기본은 꺼짐입니다.

이 PC에서 보기만 원하면 위 세션 기록 대시보드 켜기(로컬)로 충분합니다 — 이 절은 다른 기기에서 여는 고급 설정으로, 기본 127.0.0.1 대신 개인 VPN 대역(0.0.0.0+allowlist)에 노출하는 경우에만 해당합니다.

⚠️ 켜기 전 필수 — 접근 통제(allowlist) 확인

보안은 뷰어 서버가 요청자 IP를 검사해 localhost + 개인 VPN 대역만 허용하고 나머지는 403으로 거부하는 allowlist에 달려 있습니다. 이 allowlist는 뷰어(별도 공개 repo)가 구현·제공하는 속성이며 이 패키지가 강제하지 못합니다.

이 IP allowlist·CSRF 가드는 공개 뷰어 claude-session-dashboard d4482d5(2026-06-23) 이상에 반영돼 있습니다. 권장 핀 9f2bdba(2026-07-03) 이상은 기본 bind가 127.0.0.1(이 PC 전용)이라 직접 0.0.0.0을 설정하기 전에는 원격 노출 자체가 없습니다. 그 이전 커밋을 고정했거나 오설치된 경우엔 없을 수 있고, allowlist 없는 뷰어로 원격을 켜면 그 즉시 인증 없는 공개 HTTP가 됩니다. 따라서 버전과 무관하게 아래 자가검증으로 직접 확인한 뒤에만 원격을 켜세요.

자가검증 (켜기 전 필수): 같은 Wi-Fi/LAN에 있지만 VPN(Tailscale)은 끈 기기의 브라우저로 http://<호스트의 LAN IP>:포트/에 접속합니다. 명시적 403 Forbidden 으로 거부되어야 정상입니다(allowlist 동작). 화면이 그대로 보이면 allowlist가 없는 것이므로 즉시 끄세요.
⚠️ 접속 시간초과·연결거부·“페이지 없음”은 검증이 아닙니다 — NAT·라우팅 때문일 수 있어 allowlist 동작을 증명하지 못합니다(“안 열리니 안전”으로 오판 금지). 반드시 같은 LAN의 LAN IP에서 403을 직접 확인하세요. 추가로 VPN에 접속한 기기에서 http://<호스트 VPN IP>:포트/가 정상으로 열리는지도 확인합니다.

  • 절대 금지: 라우터 포트포워딩, Tailscale Funnel/Serve 공개, 클라우드 방화벽 인바운드 개방.
  • 끄기(설치된 플러그인 폴더에서; 경로는 /cockpit-doctor가 안내): bash plugin/dashboard/disable-remote.sh --apply

macOS에서 쓰기 (참고)

macOS는 1차 레퍼런스 빌드입니다. 터미널에서 Claude Code를 열고 플러그인을 설치합니다(Claude Code 설치·로그인이 먼저 필요).

  1. 1) Claude Code를 열고(터미널에서 claude) cockpit 플러그인 추가·설치:

    /plugin marketplace add https://github.com/sidoyu/cockpit
/plugin install cockpit@cc-companion

  2. 2) 설치 마법사 실행 — 동의 확인 → 충돌 검사 → dry-run → 적용(롤백 가능):

    /cockpit-setup
