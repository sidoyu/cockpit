cc-companion

cockpit — 단일 사령탑 Claude Code 환경

한 사람의 Claude Code 운영 환경을, 비개발 동료가 “마법사가 시키는 대로” 따라 하면 거의 그대로 깔 수 있게 만든 패키지입니다. 개인정보·시크릿은 전부 제거돼 있고, 각자 자기 것을 설치 마법사로 주입합니다.

이 페이지는 설치 엔진이 아니라 안내서입니다. 브라우저는 무엇도 자동으로 깔지 못하며, 아래 명령을 직접 복사·붙여넣기해서 실행합니다.

⚠️ 먼저 읽으세요 — 사용 경계

이 환경은 강력하지만 느슨하게 설정되어 있습니다. 설치 전에 아래를 이해하고 동의해야 합니다.

  • 권한 확인 생략(bypass) = 켜짐. Claude가 명령 실행 전 일일이 묻지 않고 진행합니다(대표적 파괴/비가역 명령은 금지목록으로 차단하지만 완전한 보호는 아닙니다).
  • 이중 송출. 입력 내용이 Anthropic에, 그리고 (보조 검토를 켜면) OpenAI에도 전송될 수 있습니다.
  • 원격 대시보드 = 기본 꺼짐(opt-in). 켜면 개인 VPN 안의 다른 기기에서 호스트를 조종할 수 있습니다(가장 위험).
  • 개인 PC에서, 비업무·비민감 데이터로만. 환자정보·개인정보(PII)·기밀을 절대 입력하지 마세요.
  • 안전장치는 보조일 뿐이며 최종 책임은 사용자에게 있습니다. 회사 보안정책과 충돌하면 회사 정책이 우선합니다.

무엇이 깔리나

작업 규율(행동층)

자율 진행 경계, 두 순간 자기점검, 읽기/검색 위생. 설치 마법사가 ~/.claude/CLAUDE.md로 적용.

메모리 시스템

파일 기반(md + python 훅). 세션 시작 시 상태 주입, 종료 시 기억 후보 추출, 인덱스 자동 재생성.

안전망

파괴/비가역 명령 금지목록(deny-list), 긴급정지(kill switch), 감사 로그.

보조 검토(Codex · 선택)

외부 LLM CLI를 “보이지 않는 2차 검토자”로 호출. 기본 비활성·스위치식·과금 차단.

원격 대시보드(선택)

세션 로그 대시보드를 개인 VPN 내부 기기에서 보기/조종. 기본 꺼짐.

설치 마법사

기본 사용자·메모리 디렉터리·선택 기능을 안내하고 충돌 검사·dry-run·롤백을 제공.

사전 준비 (직접 해야 하는 수동 단계)

OAuth 로그인·외부 서비스 가입은 자동화할 수 없습니다. 먼저 끝내 두세요.

Windows — WSL2 경로

Windows에서는 유닉스 자산을 별도 WSL2 배포판(기본 cc-cockpit)으로 가져와 거의 그대로 돌립니다(기존 Ubuntu를 건드리지 않음).

✅ 부트스트랩 준비됨 — 다운로드 → SHA-256 체크섬 확인 → 명시적 실행(irm … | iex 미사용 · 코드서명 없음).

  1. 0) (최초 1회) WSL2 준비 — 관리자 PowerShell에서. 이미 돼 있으면 건너뜁니다. 부트스트랩이 미설치를 감지하면 이 명령을 안내하고 멈춥니다(스스로 권한을 올리지 않음).

    wsl --install --no-distribution
wsl --update

  2. 1) 부트스트랩과 이미지를 받고, 실행 전에 검증합니다. 이 빌드는 코드서명이 없으므로(unsigned), 받은 Install-Cockpit.ps1의 SHA-256이 아래 다운로드 표의 값과 정확히 일치하는지 직접 대조하세요(무결성 보장은 이 체크섬 대조뿐):

    Get-FileHash .\Install-Cockpit.ps1 -Algorithm SHA256

    해시가 다르면 실행하지 마세요. Get-AuthenticodeSignature는 이 빌드에서 NotSigned로 나오는 것이 정상입니다(서명층 없음). 이미지(cockpit-wsl.tar.gz)의 해시는 스크립트 안에 박혀 있어, 받아온 이미지를 import 전에 자동 대조합니다(불일치 시 삭제·중단).

  3. 2) 부트스트랩 실행 — 별도 배포판으로 import(위험 기능은 전부 OFF로 출고):

    powershell -NoProfile -ExecutionPolicy Bypass -File .\Install-Cockpit.ps1

    골든 이미지 다운로드가 실패하거나 구형 환경이면 스테이지드 폴백(Install-Cockpit-Staged.ps1)을 씁니다 — 공식 베이스 rootfs를 검증해 import 후 라이브 프로비저닝. 통째 삭제는 wsl --unregister cc-cockpit.

  4. 3) WSL(cc-cockpit) 안에서 Claude Code 로그인 후 cockpit 플러그인 설치:

    claude
/plugin marketplace add https://github.com/sidoyu/cockpit
/plugin install cockpit@cc-companion
/cockpit-setup

macOS — 네이티브 경로

macOS는 1차 레퍼런스 빌드입니다. 터미널에서 Claude Code를 열고 플러그인을 설치합니다.

  1. 1) Claude Code를 열고(터미널에서 claude) cockpit 플러그인 추가·설치:

    /plugin marketplace add https://github.com/sidoyu/cockpit
/plugin install cockpit@cc-companion

  2. 2) 설치 마법사 실행 — 동의 확인 → 충돌 검사 → dry-run → 적용(롤백 가능):

    /cockpit-setup

설치 후 — 점검·끄기·지우기

상태 점검

언제든 설치 상태·원격 ON/OFF·보조 검토 ON/OFF를 확인:

/cockpit-doctor

긴급 정지

자동 진행을 즉시 멈춤(파일 하나 생성):

touch ~/.claude/CC_KILL_SWITCH

보조 검토 끄기

OpenAI 송출을 멈춤(스위치 파일 삭제):

rm ~/.claude/codex_enabled

원격 끄기

먼저 미리보기, 그다음 실제 비활성화(설치된 플러그인 폴더에서 실행):

bash plugin/dashboard/disable-remote.sh
bash plugin/dashboard/disable-remote.sh --apply

경로가 안 맞으면 “file not found”가 납니다 — 정확한 절대경로는 /cockpit-doctor가 안내합니다.

전체 제거는 /plugin uninstall + 설치 마법사의 롤백 안내를 사용합니다. 자세한 절차는 거버넌스 문서 6장.

원격 대시보드 (고급 · 선택 · 기본 꺼짐)

휴대폰·노트북 등 같은 개인 VPN(예: Tailscale)에 묶인 다른 기기에서 세션 로그를 보거나 세션을 조종하는 선택 기능입니다. 이 패키지에서 가장 위험한 기능이라 기본은 꺼짐입니다.

⚠️ 켜기 전 필수 — 접근 통제(allowlist) 확인

보안은 뷰어 서버가 요청자 IP를 검사해 localhost + 개인 VPN 대역만 허용하고 나머지는 403으로 거부하는 allowlist에 달려 있습니다. 이 allowlist는 뷰어(별도 공개 repo)가 구현·제공하는 속성이며 이 패키지가 강제하지 못합니다.

이 IP allowlist·CSRF 가드는 공개 뷰어 claude-session-dashboard d4482d5(2026-06-23) 이상에 반영돼 있습니다. 그 이전 커밋을 고정했거나 오설치된 경우엔 없을 수 있고, allowlist 없는 뷰어로 원격을 켜면 그 즉시 인증 없는 공개 HTTP가 됩니다. 따라서 버전과 무관하게 아래 자가검증으로 직접 확인한 뒤에만 원격을 켜세요.

자가검증 (켜기 전 필수): 같은 Wi-Fi/LAN에 있지만 VPN(Tailscale)은 끈 기기의 브라우저로 http://<호스트의 LAN IP>:포트/에 접속합니다. 명시적 403 Forbidden 으로 거부되어야 정상입니다(allowlist 동작). 화면이 그대로 보이면 allowlist가 없는 것이므로 즉시 끄세요.
⚠️ 접속 시간초과·연결거부·“페이지 없음”은 검증이 아닙니다 — NAT·라우팅 때문일 수 있어 allowlist 동작을 증명하지 못합니다(“안 열리니 안전”으로 오판 금지). 반드시 같은 LAN의 LAN IP에서 403을 직접 확인하세요. 추가로 VPN에 접속한 기기에서 http://<호스트 VPN IP>:포트/가 정상으로 열리는지도 확인합니다.

  • 절대 금지: 라우터 포트포워딩, Tailscale Funnel/Serve 공개, 클라우드 방화벽 인바운드 개방.
  • 끄기(설치된 플러그인 폴더에서; 경로는 /cockpit-doctor가 안내): bash plugin/dashboard/disable-remote.sh --apply
