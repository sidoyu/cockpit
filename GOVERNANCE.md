# 거버넌스 경계 문서 (GOVERNANCE)

> 이 문서는 **읽고 동의한 사람만** 이 패키지를 설치·사용한다는 전제로 배포된다.
> 설치 마법사는 첫 실행 때 이 문서의 동의 확인을 요구한다.
> 문서 버전: v0.1 (초안) · 최종 결정권자: 배포자(패키지 제공자)

---

## 0. 한 문단 요약 (먼저 읽으세요)

이 패키지는 **강력하지만 느슨하게 설정된** Claude Code 환경입니다. ⓐ 기본으로 Claude가 **명령 실행 전 일일이 묻지 않고 진행**(bypass)하고, ⓑ **외부 기기(휴대폰·다른 브라우저)에서 원격 조종**이 가능하며 — claude.ai Code 탭 원격 조종(Remote Control)은 **기본 ON**(아웃바운드 HTTPS 전용·수신 포트 없음·본인 계정 한정), 자체호스팅 대시보드는 **명시 설정 시에만 켜지는 opt-in** —, ⓒ 입력 내용이 모델 응답을 위해 **Anthropic에 전송**되고, 일부 기능(메모리 자동 추출·선택적 대시보드 요약)은 **추가 외부 송신**을 수반할 수 있습니다(3장). 편의를 위해 안전 장치(금지목록·긴급정지·감사로그)를 내장했지만, 이것들은 **사용자의 판단을 대신하지 않습니다.** 따라서 이 환경은 **개인 PC에서, 비업무·비민감 데이터로만** 사용해야 합니다. 회사 환자정보·의료기록·개인정보(PII)·기밀을 이 환경에 절대 입력하지 마세요.

## 1. 적용 범위

- **대상**: 이 패키지를 설치하는 모든 사용자(주로 비개발 동료).
- **무엇에 적용되나**: 이 패키지로 설치된 Claude Code 환경 전체 — 행동 규율(플러그인), 메모리 시스템, 원격 제어/대시보드, 그리고 설치 마법사가 적용하는 설정.
- **무엇에 적용 안 되나**: 사용자가 이 패키지와 무관하게 별도로 설치·운영하는 다른 도구. 단, 같은 PC를 공유하면 영향이 번질 수 있으니 2장(경계 규칙)을 따른다.

## 2. 경계 규칙 (반드시 지킬 것)

### 2.1 개인 PC 전용
- **개인 소유 PC에만 설치**한다. 회사가 관리(MDM/자산관리)하는 PC, 임상/진료 시스템에 연결된 PC, 공용 PC에는 설치하지 않는다.
- 이 패키지는 **회사 공식 IT 배포물이 아니다.** 동료가 자신의 운영 환경을 공유한 **개인 학습용 도구**이며, 회사 보안정책과 별개다. 회사 정책과 충돌하면 회사 정책이 우선한다.

### 2.2 비업무·비민감 데이터 전용
- **회사 업무 데이터·환자정보·의료기록·계약·인사·재무 등 민감/기밀 정보를 입력하지 않는다.**
- 용도는 **개인 학습·실험·개인 프로젝트**다. 업무 자동화가 필요하면 회사가 공식 승인·관리하는 경로를 사용한다.

### 2.3 개인정보(PII)·의료정보 금지
- 실명·주민등록번호·연락처·주소·환자식별정보·진단/처방 등 **개인을 식별할 수 있는 정보를 프롬프트·파일·메모리에 넣지 않는다.**
- 이미 넣었다면 6장(비활성화·삭제)으로 제거하고, 외부 전송이 일어났을 수 있음을 인지한다(아래 3장).

### 2.4 안전장치는 보조일 뿐
- 내장된 금지목록·긴급정지·감사로그는 **명백한 사고를 줄이기 위한 그물**이지, 완전한 보호가 아니다. 최종 책임은 사용자에게 있다.

## 3. 외부 송출 고지 (데이터가 어디로 가나)

- **Anthropic**: Claude Code를 쓰는 동안 입력한 프롬프트·코드·파일 일부가 모델 응답 생성을 위해 Anthropic에 전송된다(Claude 서비스의 기본 동작).
- **메모리 자동 추출(별도 API 송신)**: 세션 종료 시 '기억할 만한 정보' 추출을 위해 **대화 일부(시크릿 레닥션 후)가 Anthropic의 별도 경량 모델 API로 전송**된다 — Claude Code 본 기능과 **구분되는 추가 송신**이다. 이 송신은 **두 조건을 모두 만족할 때만** 일어난다: ① 설치 마법사(`/cockpit-setup`)를 완료해 동의했을 것(`STATE_DIR/setup_complete` 마커가 게이트 — 플러그인만 깔린 동의 전 상태에서는 키가 있어도 송신 안 함), ② 사용자의 `ANTHROPIC_API_KEY` 가 설정돼 있을 것. 둘 중 하나라도 없으면 추출만 비활성(나머지 기능 정상)이고, 끄려면 키를 비우거나 설치를 롤백한다(마커 제거 → 다시 OFF). (저장 직전에도 결과를 한 번 더 레닥션하지만, 민감정보 비입력이 유일하게 확실한 보호다.)
- **원격 제어/대시보드**: 외부 기기에서 조종·관찰하는 기능을 켜면, 세션 내용이 사용자의 네트워크(예: Tailscale 같은 개인 VPN)를 통해 사용자의 다른 기기로 전달된다. 제3자 클라우드에 별도 저장하지 않도록 구성하는 것을 기본으로 한다.
- **대시보드 부가 송신(선택 — 켠 경우에 한함)**: 원격 대시보드의 주기 변환 cron 에는 **선택 기능 두 가지**가 있다. ① **AI 제목 요약**을 켜면(`CC_DASH_SUMMARY_KEY_ENV` 설정) **세션 본문 발췌가 별도 LLM 공급자로 송출**된다 — 켜지 않으면 송출 없음. ② **healthcheck ping**을 설정하면(`CC_DASH_HEALTHCHECK_URL`) 외부 모니터링 URL로 **메타데이터(완료 신호)만** 전송한다(세션 내용 아님). 둘 다 기본은 **빈 값=꺼짐**이고, 켜는 경우 위 2.3 의 민감정보 금지가 그대로 적용된다.
- **결론**: 민감 데이터를 넣지 않는 것이 유일하게 확실한 보호다(2.3).

## 4. 기본값으로 켜진 위험 기능과 그 이유

| 기능 | 기본값 | 왜 켜는가 | 내장 완충 |
|------|--------|-----------|-----------|
| 권한 확인 생략(bypass) | ON | 비개발자가 매번 확인 팝업에 막히지 않게 | 파괴/비가역 명령 **금지목록**으로 자동 차단(5장) |
| 원격 조종(claude.ai Remote Control) | **ON**(이미지 배포본·`remoteControlAtStartup`) | 터미널이 어려운 동료가 claude.ai Code 탭/모바일에서 자기 로컬 세션을 잇게 | **아웃바운드 HTTPS 전용·수신 포트 없음**(Anthropic 공식 기능). 본인 계정으로만 접근·단명 자격증명·끄기 `/config`(아래 ※) |
| 자체호스팅 대시보드(읽기전용 뷰어) | **opt-in(기본 OFF·아래 ※2)** | 진행 관찰(원격조종과 별개) | 개인 VPN 내부 한정(localhost+VPN 대역만 허용 — **뷰어가 allowlist 를 구현했을 때만 성립**, ※3)·`disable-remote`·감사로그 |
| 메모리 외부송신(egress) | **OFF**(첫 실행 동의 게이트) | 끄면 메모리는 로컬에서 정상 동작 | `--enable-memory-egress`+동의 둘 다 있어야 켜짐(bypass 동의와 분리) |

이것들은 **의도적으로 수용한 리스크**다. 편의를 얻는 대신, 위 2~3장의 경계를 지키는 것이 조건이다.

> ※ **원격 조종(claude.ai Remote Control)**: 로컬 Claude Code 세션이 **나가는(아웃바운드) HTTPS 로만** Anthropic
> API 에 등록·폴링하며 **기기에 수신 포트를 열지 않는다** → VPN/포트 개방 불필요, "켜져 있어도 외부 수신 노출 0".
> 같은 Claude 계정으로 claude.ai/code 또는 모바일 앱에 로그인한 사람만 그 세션을 보고 조종할 수 있다(본인 계정).
> 끄기: `claude /config` → "Enable Remote Control for all sessions" → `false`(또는 settings `remoteControlAtStartup:false`).
> 전제: claude.ai OAuth 로그인(API 키 인증 불가). ⚠️ **`ANTHROPIC_API_KEY` 가 환경에 설정돼 있으면 Remote Control 이
> 거부**되므로, 메모리 외부송신용 키는 `ANTHROPIC_API_KEY` 가 아니라 **`ANTHROPIC_API_KEY_FOR_SCRIPTS`** 로 둔다.
> Team/Enterprise 플랜은 Owner 가 admin 에서 토글을 켜야 한다(개인 Pro+ 는 해당 없음).
>
> ※2 **자체호스팅 대시보드는 위 원격 조종과 별개**다. 이 빌드에선 설치 마법사 자동 배선이 없어 **명시 설정해야만 켜진다**(옵트인).
>
> ※3 **“localhost+VPN 대역만 허용”은 뷰어가 IP allowlist 를 구현했을 때만 성립한다(이 패키지가 강제하지 못함).**
> allowlist·CSRF 가드는 공개 뷰어 `claude-session-dashboard` **`d4482d5`(2026-06-23) 이상에 반영**돼 있고,
> **권장 핀 `9f2bdba`(2026-07-03) 이상은 기본 bind 가 `127.0.0.1`(로컬 전용)**이라 명시적으로 `0.0.0.0` 을
> 설정하기 전에는 원격 노출 자체가 없다(`CC_DASH_BIND`·idle-exit 지원 포함).
> 그 이전 커밋·오설치 버전엔 없을 수 있고, allowlist 없는 뷰어로 원격을 켜면 그 즉시 **인증 없는 공개 HTTP** 가 된다.
> 따라서 버전과 무관하게 **원격을 켜기 전, `plugin/dashboard/README.md` 의 자가검증
> (같은 LAN·VPN 끈 기기에서 `403` 확인)을 반드시 먼저 수행**하고, 403 이 확인되지 않으면 원격을 켜지 않는다.

## 5. 내장 안전망 (무엇이 막아주나)

1. **파괴/비가역 명령 금지목록(deny-list)**: 다음 부류의 **흔한 형태**를 bypass 상태여도 자동 차단한다 — 루트/홈/시스템 경로나 와일드카드(`*`)를 지우는 `rm -rf`, `find … -delete`/`-exec rm`, `rsync --delete`, `xargs rm -rf`, 시스템 파일 `truncate`, 데이터베이스 드롭, 강제 푸시(force push), 파이프-투-셸(`curl … | sh`), 디스크 덮어쓰기, 계정/리소스 삭제 예시 등. **단 이것은 backstop 이며 완전하지 않다** — 예컨대 경로를 변수에 담아 지우거나(`d=…; rm -rf "$d"`) 새로운 우회 형태는 잡지 못한다(`rm` 차단은 의도적으로 루트/홈/시스템/와일드카드 대상에 한정). **1차 방어는 Claude 의 판단 + '멈춰 질문' 규율 + kill switch** 이고, 금지목록은 그 뒤를 받치는 그물일 뿐이다. (정확한 목록은 `plugin/safety/deny-list` 참조.)
2. **긴급 정지(kill switch)**: 한 번의 동작으로 자동 진행을 즉시 멈출 수 있다(6장).
3. **감사 로그(audit log)**: 차단된 시도와 민감 동작이 로컬에 기록되어 사후 확인이 가능하다. (로그 자체에 민감정보를 남기지 않도록 레닥션 적용.)

이 안전망은 **차단 위주**다. 새로운 위험 패턴까지 모두 막지는 못한다.

## 6. 비활성화·축소·삭제 절차

> 정확한 명령·경로는 설치 마법사 완료 화면과 README에 함께 표시된다. 아래는 개념 절차.

- **자동 진행만 끄기(가장 빠른 완화)**: 설정에서 권한 모드를 기본(`ask`)으로 되돌린다 → 이후 Claude가 다시 일일이 확인한다.
- **원격 제어 끄기**: 원격 대시보드(다른 기기에서 호스트 조종)는 **선택 기능이며 기본 비활성**이다. 켜져 있다면 다음으로 끈다.
  - **미리보기(무엇을 멈출지)**: `bash plugin/dashboard/disable-remote.sh` (**설치된 플러그인 폴더에서 실행** — 경로가 안 맞으면 `cockpit-doctor` 가 안내하는 절대경로를 쓴다)
  - **실제 비활성화**: `bash plugin/dashboard/disable-remote.sh --apply` — 서버 프로세스 중지(포트 탐지 lsof→ss 폴백) + 자동시작 해제((macOS) launchd · (Linux/WSL) systemd --user 유닛이 있으면) + 포트 닫힘 확인 + VPN(Tailscale 등) 접근 차단·(WSL/Windows) `netsh portproxy` 중계 제거·재활성화 안내를 수행한다(멱등).
  - **상태 확인**: `cockpit-doctor`(= `setup.py doctor`)가 "원격 대시보드 ON/OFF"를 **OS별 정직 탐지**(자동시작: macOS=launchd·Linux/WSL=systemd --user 유닛 + 포트 LISTEN 여부)해 보고한다. WSL 에서 포트가 열려 있으면 Windows 호스트 localhost 노출 경고도 함께 낸다.
  - **접근 경계**: 켜져 있어도 대시보드는 **localhost + 개인 VPN 대역(기본 Tailscale `100.64.0.0/10`)만 허용**하고 나머지는 거부하는 것을 목표로 한다(공개망·LAN 직접 접속 차단). **단 이 차단은 뷰어가 allowlist 를 구현했을 때만 성립**하며(4장 ※3), 공개 뷰어 `d4482d5`(2026-06-23) 이상에 반영돼 있다(이전·오설치 버전엔 없을 수 있음; 권장 핀 `9f2bdba`(2026-07-03) 이상은 기본 bind `127.0.0.1` = 옵트인 전 원격 노출 자체가 없음) → **버전과 무관하게 원격을 켜기 전 자가검증(같은 LAN·VPN 끈 기기에서 `403` 확인) 필수**. 공개망 노출 구성으로 바꾸지 말 것. 상세: `plugin/dashboard/README.md`.
- **긴급 정지(kill switch)**: 긴급정지 파일을 생성한다(예: `touch ~/.claude/CC_KILL_SWITCH`) → 자동 진행이 즉시 멈춘다.
- **전체 제거(uninstall)**: 플러그인 제거(`/plugin uninstall ...`) + 마법사가 설치한 설정/파일 되돌리기(마법사의 롤백 안내 사용). 메모리·로그는 사용자가 직접 삭제할 수 있다.
  - **Windows/WSL**: cockpit 은 **별도 WSL 배포판**(기본 `cc-cockpit`)으로 설치되므로, 통째 제거는 Windows PowerShell 에서 `wsl --unregister cc-cockpit` 한 줄이다(비개발자는 **원터치 `Cockpit-Uninstall.cmd` 더블클릭** — 이름 재입력 확인 후 같은 동작) — **기존 Ubuntu 등 다른 배포판은 건드리지 않는다.** 문제 복구/재설치는 `Cockpit-Repair.cmd`(재설치=배포판 내부 데이터 소실, 사전 백업 권장). 설치기는 `irm…|iex` 를 쓰지 않고 *다운로드 → SHA-256 체크섬 검증 → 명시 실행* 순서이며(이 배포본은 코드서명 없음 — 무결성은 체크섬 대조), 자체적으로 관리자 권한을 올리지 않는다(WSL 미설치 시 안내만). 상세: `windows/README.md`.

## 7. 책임 경계

- 이 패키지는 **있는 그대로(as-is)** 제공된다. 제공자는 동료의 호의로 자신의 환경을 공유할 뿐, 무결성·적합성·손해에 대한 보증을 하지 않는다.
